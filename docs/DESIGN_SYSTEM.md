# HUAZAIDESIGN 统一设计规范与操作逻辑

本文档是 HUAZAIDESIGN 后续界面修改的统一基准。新增功能、修复 UI、调整节点样式时，优先遵守本文档；如现有实现冲突，以本文档作为重构目标。

## 1. 产品定位

HUAZAIDESIGN 是面向 AI 图像、视频、文字、音频工作流的画布工具，不是营销页，也不是传统表单系统。

界面目标：

- 画布优先：用户第一眼看到的是内容、节点和窗口，不是装饰。
- 操作明确：单击选中，双击编辑，右键打开上下文菜单，拖拽移动。
- 类 Figma 体验：画布、图层、属性、工具栏、快捷键和文本编辑逻辑尽量接近主流设计软件。
- 少说明，多直接操作：界面不写大段教程，只保留必要标签、状态和错误提示。
- 深浅色一致：所有功能在深色和浅色下必须可读、可点、可辨识。

## 2. 视觉系统

### 2.1 品牌

- 品牌名：`HUAZAIDESIGN`
- 主 Logo：黑色圆角底 + 银白莫比乌斯环 + 面性播放三角。
- 所有入口统一使用 `/liang.svg` 或同源导出的 Electron 图标资源。
- 不允许同一界面混用旧图标、文字 Logo、不同风格的临时图标。

### 2.2 色彩

基础色只承担层级，不承担装饰。

深色模式：

- 画布背景：`#2B2B2B`
- 点阵：`#D6D6D6`
- 主面板背景：`rgba(9, 9, 11, 0.94)` 或 `var(--hz-surface-strong)`
- 次级表面：`rgba(255,255,255,0.055)`
- 主文字：`#F4F3EE`
- 次级文字：`rgba(244,243,238,0.72)`
- 弱提示文字：`rgba(244,243,238,0.46)`
- 边框：`rgba(255,255,255,0.12)`

浅色模式：

- 画布背景：`#E6E6E6`
- 点阵：`#C7C7C7`
- 主面板背景：`rgba(255,255,255,0.96)`
- 次级表面：`rgba(13,13,13,0.055)`
- 主文字：`#111111`
- 次级文字：`rgba(17,17,17,0.68)`
- 弱提示文字：`rgba(17,17,17,0.44)`
- 边框：`rgba(13,13,13,0.12)`

节点类型强调色只用于识别，不允许大面积铺色：

- 上传 / 资源：绿色系
- 图像：琥珀系
- 视频：玫红系
- SD2.0：紫红系
- 文字：蓝灰系
- 音频：紫色系
- 工具：橙色系
- 画板：中性琥珀或灰色

### 2.3 圆角、阴影、边框

- 工具按钮：圆形或 8px 以下圆角。
- 浮层、菜单、弹窗：8-12px 圆角。
- 不使用大圆角卡片堆叠，不做卡片套卡片。
- 画布内容节点不能因为选中而出现厚重外框。
- 图像、视频、上传图像结果默认不加外框，只用背景差异和顶部工具条区分状态。
- 阴影只用于浮层和面板，不用于普通画布内容。

### 2.4 字体

默认字体：

```css
Inter, system-ui, -apple-system, "PingFang SC", "Microsoft YaHei", sans-serif
```

文字原则：

- 工具栏按钮优先图标，不写长文字。
- 面板标题 12-14px。
- 标签 10-11px。
- 输入内容 11-13px。
- 画布文字节点由用户字号控制，不受 UI 标签规范限制。
- 不使用负字距。

## 3. 布局规范

### 3.1 启动页

启动页只承担项目入口：

- 顶部中央显示 Logo、品牌名、状态。
- 主操作为“新建项目”。
- 最近使用最多展示 5 个，更多面板最多 20 个。
- 最近使用卡片支持：
  - 单击打开
  - 右上角关闭当前预览
  - 右键菜单：打开、关闭、删除
- 关闭只隐藏当前页面预览，不删除画布。
- 删除必须二次确认。

### 3.2 画布页

画布页结构固定：

- 顶部栏：Logo 返回启动页、菜单按钮、项目名、版本、后端状态、保存、共享文件、云盘、设置、主题。
- 中央：ReactFlow 无限画布。
- 底部：核心工具快捷入口。
- 右侧：属性面板 + 窗口顺序。
- 浮层：右键菜单、连线候选、媒体设置、系统通知。

顶部 Logo 操作：

- 单击 Logo 返回启动页。
- 返回前保存当前画布。
- 不删除画布，不清空项目数据。

底部工具栏：

- 只放高频操作。
- 按钮使用图标，tooltip 说明功能。
- 节点新增默认落在当前视口中心。

右侧属性面板：

- 展示选中数量。
- 多选时显示对齐、图层相关操作。
- 选中文字时显示文字设置。
- 底部始终显示窗口顺序列表。

## 4. 画布操作逻辑

### 4.1 基础选择

- 单击节点：选中节点。
- Shift / Ctrl / Meta + 单击：多选。
- 框选：多选节点，但不自动弹出菜单。
- 单击画布空白：取消选择。
- 右键节点：打开节点上下文菜单。
- 右键多选区域：打开多选上下文菜单。
- 右键画布空白：打开快速新增菜单。

禁止行为：

- 普通单击不能触发素材拖拽。
- 多选完成不能自动弹出菜单。
- 鼠标松开后节点或素材不能继续跟随鼠标。

### 4.2 移动与拖拽

节点移动：

- 普通拖拽节点移动位置。
- 拖拽时显示对齐辅助线。
- 松开后保存位置。
- 自动生成的输出节点被用户移动后记录 `userMoved`，后续自动排布不得覆盖。

素材拖拽：

- Ctrl / Meta + 从图像、视频、音频素材上拖拽，才启动跨节点素材拖拽。
- 普通鼠标拖动媒体区域，只移动节点或操作媒体控件。
- 所有媒体元素必须 `draggable={false}`，并阻止浏览器原生 drag。

### 4.3 连线

- Handle 只在节点选中时显示。
- 从 source 拖到 target 建立连接。
- 不兼容端口不允许连接。
- 拖到空白处显示候选节点菜单。
- Shift + Handle 拖拽用于批量重连。

### 4.4 图层 / 窗口顺序

窗口顺序指画布渲染层级：

- 最上层显示在列表顶部。
- 最底层显示在列表底部。
- 列表显示所有可见窗口节点，排除临时节点、隐藏输出节点、废弃节点。

图层操作：

- 置顶
- 上移
- 下移
- 置底

可排序节点：

- 图像生成
- 上传图像
- 文字
- 视频
- SD2.0
- 画板
- 其他明确作为画布窗口存在的节点

点击窗口顺序项：

- 选中对应节点。
- 画布定位到节点中心。
- 不打开节点设置菜单，除非该节点自身定义了单击打开菜单。

## 5. 节点规范

### 5.1 通用节点

通用节点由三部分组成：

- 内容区
- 选中状态
- 设置菜单

规范：

- 单击选中。
- 双击只用于编辑内容或进入专用编辑模式。
- 右键打开上下文菜单。
- 运行状态必须显示 loading / progress / error。
- 错误信息必须可复制或完整可读。
- 深色模式下 label、input、select、option、textarea、按钮都必须可见。

### 5.2 图像生成节点

显示逻辑：

- 未生成：显示图像占位、生成状态和必要提示。
- 已生成：只显示图像内容，不加厚重外框。
- 深色模式下用背景色区分区域，不用描边包裹图像。
- 顶部左侧显示比例、分辨率或尺寸信息。
- 顶部右侧显示缩放、重置、下载、设置等图标。

操作：

- 单击节点：选中并显示工具条和设置菜单。
- 双击图像：进入编辑。
- Ctrl / Meta + 拖图像：作为素材拖到其他节点。
- 下载按钮每次点击都必须有效。

### 5.3 视频节点

显示逻辑与图像一致：

- 视频区域外显示顶部信息和工具条。
- 视频控件不应阻止节点选中。
- 点击视频边缘、中心、空白区域都应能选中节点。

操作：

- 单击选中并显示菜单。
- 视频播放控件保持浏览器原生行为。
- Ctrl / Meta + 拖视频：作为素材拖到其他节点。

### 5.4 SD2.0 节点

SD2.0 按视频节点处理，但设置项属于 Seedance：

- 单击任意区域选中。
- 设置菜单必须容易打开。
- 视频区域不得拦截节点选择。
- 顶部左侧显示比例、时长、分辨率。
- 顶部右侧显示缩放、重置、下载、设置。

### 5.5 上传节点

上传节点分两种状态：

未上传：

- 显示轻量上传区域。
- 支持点击选择、拖拽上传。
- 可接受图像、视频、音频。

已上传图像：

- 逻辑与图像生成结果一致。
- 不显示额外画面外界面框。
- 顶部左侧显示文件名、尺寸或大小。
- 顶部右侧显示编辑、下载、替换、清空。

已上传视频 / 音频：

- 保留原生媒体控件。
- 补齐深色模式可见性。
- 保留下载、替换、清空。

PNG 透明底图：

- 导入和上传不能做格式转换。
- 不能铺白底或黑底。
- 图像节点显示时背景透明。

### 5.6 文字节点

文字节点只用于标注和排版，不参与连线。

操作：

- 单击选中。
- 双击编辑。
- 输入后按文本本身显示。
- 选中时显示贴合文字的轻量边界。
- 不显示四角缩放光标。
- 不显示连接 Handle。

属性：

- 字体
- 字号
- 字重
- 行高
- 字距
- 颜色
- 对齐

字号、字体、颜色变化后，文字框必须重新测量并贴合文本。

## 6. 菜单与工具条

### 6.1 媒体工具条

媒体工具条统一放在内容区域外部上方：

- 左侧：尺寸、比例、分辨率、文件信息。
- 右侧：功能图标。

常用图标顺序：

1. 缩小
2. 重置缩放
3. 放大
4. 下载
5. 设置或编辑

图标必须有 tooltip。

### 6.2 设置菜单

设置菜单统一规则：

- 从节点下方弹出。
- 宽度 340-360px。
- 最大高度不超过视口 70%。
- 内部滚动。
- 深色模式必须使用统一变量，不写死不可见颜色。

表单控件：

- label 10px。
- input/select/textarea 11-12px。
- 圆角 6-8px。
- 边框弱化。
- focus 状态可见。

### 6.3 右键菜单

右键菜单只放上下文动作：

- 打开 / 编辑
- 复制
- 删除
- 置顶 / 上移 / 下移 / 置底
- 分组 / 取消分组

不在右键菜单中放长期设置表单。

## 7. 文件与路径操作

### 7.1 导入

导入按钮支持：

- JSON 画布文件：替换当前画布，需要确认。
- PNG 图片：作为上传图像节点加入当前画布，不替换画布。

后续如增加格式：

- JPG / WEBP：作为上传图像节点。
- MP4 / WEBM：作为上传视频节点。
- MP3 / WAV：作为上传音频节点。

### 7.2 下载

下载逻辑：

- 优先使用用户设置的下载目录。
- 未设置下载目录时弹出选择目录。
- 下载失败必须说明原因。
- 第二次点击下载必须重新触发，不受缓存 URL 影响。

### 7.3 共享文件夹和云盘

- 已配置共享文件夹：点击直接打开本地目录。
- 路径不存在：提示错误，不跳回设置。
- 未配置共享文件夹：打开设置。
- 已配置云盘：点击打开外部链接。
- 云盘地址无协议时自动补 `https://`。
- 未配置云盘：打开设置。

## 8. 深色模式规则

所有新增组件必须检查深色模式：

- 文字不可低于 45% 白色透明度，除非只是装饰。
- input/select/textarea 背景和文字必须对比明显。
- option 必须设置背景色和文字色。
- 图像、视频、SD2.0 菜单不能使用硬编码 `text-white/30` 作为主要信息。
- 图像内容区域不能靠黑底 + 黑字表达状态。
- 错误提示用红色，但需要暗底对比。

禁止：

- 同一组件同时使用多个不相关色系。
- 深色下透明度过低导致不可读。
- 依赖 hover 才显示关键操作。

## 9. 状态与反馈

通用状态：

- idle：普通状态。
- selected：选中状态。
- hover：轻微背景变化。
- active：按钮当前开启。
- loading：转圈或进度。
- success：短暂成功提示。
- error：明确错误文本。
- disabled：降低透明度并禁止点击。

通知：

- 下载成功：显示保存路径，可打开目录。
- 下载失败：显示失败原因。
- 更新可用：显示版本和入口。
- 后端异常：顶部状态显示异常，具体错误进入日志或弹窗。

## 10. 快捷键

基础快捷键：

- `Ctrl / Meta + Z`：撤销
- `Ctrl / Meta + Shift + Z` 或 `Ctrl + Y`：重做
- `Ctrl / Meta + C`：复制
- `Ctrl / Meta + V`：粘贴
- `Ctrl / Meta + Shift + V`：带连接粘贴
- `Ctrl / Meta + D`：复制并粘贴
- `Delete / Backspace`：删除选中
- `Ctrl / Meta + A`：全选
- `Ctrl / Meta + G`：分组
- `Ctrl / Meta + ]`：上移
- `Ctrl / Meta + [`：下移
- `Ctrl / Meta + Shift + ]`：置顶
- `Ctrl / Meta + Shift + [`：置底

快捷键不应在 input、textarea、contenteditable 中抢占文本编辑行为。

## 11. 实施约束

新增或修改 UI 时必须满足：

- 图标优先使用 `lucide-react`。
- 不新增独立色彩系统，优先使用 `theme-grok.css` 变量。
- 不新增另一套节点选择逻辑，复用 ReactFlow 选中状态。
- 不新增另一套图层排序逻辑，复用 `reorderLayerNodes`。
- 媒体节点普通单击只做选择，不启动素材拖拽。
- Ctrl / Meta 拖拽素材必须有统一兜底释放逻辑。
- 文案必须短，不写说明型段落进主界面。
- 任何深色模式改动都要同时检查浅色模式。

## 12. 验收清单

每次 UI 改动至少检查：

- 深色模式可读。
- 浅色模式可读。
- 单击、双击、右键、拖拽行为符合规范。
- 多选不会自动弹菜单。
- 图层顺序可理解。
- 下载、导入、上传不会被缓存或原生拖拽干扰。
- 文字节点颜色、字号、字体修改后即时生效。
- 安装包版本和应用内版本一致。
